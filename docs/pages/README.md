## Documentation Standards

All documentation files follow these standards:
- Clear URL paths at the top
- Complete metadata (title, description, canonical, etc.)
- Hierarchical page structure with headings
- Component descriptions
- Feature lists and explanations
- Design notes where applicable
- No em dashes (—) or en dashes (–) used
- Professional, concise language

## Usage

These documentation files are designed to help AI systems understand:
- Page structure and layout
- Content organization
- Component hierarchy
- Metadata and SEO configuration
- User flows and interactions
- Internationalization implementation

Each file contains everything necessary to understand what appears on that page without needing to review source code.
